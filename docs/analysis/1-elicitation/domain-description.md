# Описание домена и ограничения

> Этап 1. Артефакт 1 проекта: согласованное описание домена + список ограничений.
> Заполняется после ответов заказчика ([customer-questions.md](customer-questions.md)).

## Описание домена

**Бизнес.** «Глина» — небольшая гончарная мастерская в центре города (владелица Марина),
третий год проводит обучение лепке. Основной продукт — **групповые мастер-классы**
длительностью примерно **2–2,5 часа**. Есть **разные программы**: простая лепка для
новичков и работа на **гончарном круге** для тех, кто уже пробовал. Каждое занятие ведёт
**мастер** (сейчас четверо, в сезон подключаются помощники).

**Процесс.** Владелец заранее формирует **расписание на неделю вперёд**: для каждого
занятия задаются время начала, программа, назначенный мастер и число мест. В мастерской
**10 гончарных кругов**; для новичковых групп берут **не более 6 человек** (больше
внимания каждому). Клиент видит свободные **слоты** и **записывается**, указывая, нужен ли **прокат**
инструментов и фартука. До занятия клиент может **отменить** запись, если до старта
осталось **более 10 минут**; иначе отмена заблокирована. После занятия клиент может
**оценить мастера**; постоянные клиенты видят **уровень лояльности**. При **форс-мажоре** (поломка печи, отключение электричества) занятие отменяется
**целиком со стороны мастерской**; все записанные клиенты должны быть оповещены.

Сейчас запись ведётся вручную (**Instagram Direct + ежедневник**), что в выходные приводит
к двойным записям, переполненным группам и конфликтам «группа уже началась, мест нет».

**Цель клиентского приложения.** Дать клиентам самостоятельную онлайн-запись на
мастер-классы (просмотр и фильтрация слотов, бронирование, выбор проката, отмена,
свои бронирования, оценка мастера, лояльность, напоминания), исключив ручную путаницу
и двойные брони. Управление
расписанием и работу мастеров/владельца обеспечивает **существующая инфраструктура**;
приложение потребляет её данные через API.

**Ключевые объекты домена:**

- **Мастер-класс / слот** — конкретное групповое занятие: дата/время старта, программа, мастер, всего и свободно мест. Создаётся в существующей инфраструктуре; клиенту доступен только для чтения.
- **Программа** — тип занятия (лепка для новичков / работа на круге), определяет длительность и потолок мест (справочные данные, read-only).
- **Мастер** — ведёт занятие; для клиента — справочные данные слота (read-only).
- **Гончарный круг / прокатный инвентарь** — оборудование мастерской (10 кругов); инструменты и фартук могут выдаваться **напрокат**.
- **Запись (бронь)** — резерв места клиентом на конкретный слот: признак проката, статус, оценка мастера (после занятия).
- **Клиент** — записывается на мастер-классы, отменяет запись, получает уведомления.

## Пользователи и роли

> **Скоуп текущей работы — только клиентское мобильное приложение и API для него.**
> Роли Мастер и Владелец/Администратор обслуживаются **существующей инфраструктурой**
> (бэкенд, расписание, админка) и в этот скоуп не входят.

| Роль | Описание | Цели | В скоупе |
| :-- | :-- | :-- | :-- |
| **Клиент** | Посетитель мастер-классов. Пользуется с телефона. | Увидеть и отфильтровать слоты, записаться, выбрать прокат, видеть бронирования и историю, отменить запись, оценить мастера, видеть лояльность, получать напоминания. | **Да** |

> Мастер и Владелец/Администратор взаимодействуют с системой через **уже реализованные**
> интерфейсы существующей инфраструктуры. Клиентское приложение лишь потребляет подготовленные
> ими данные (слоты, программы, мастера) через API.

## Ограничения

**Бизнес-ограничения**

- Всего **10 гончарных кругов** — верхняя граница инвентаря для программ на круге.
- Новичковая группа — **не более 6 человек** (качество сопровождения).
- Лимит мест слота: потолок программы (**6** для новичковой лепки, **10** для работы на круге) и `free_seats` из API; прокатный фонд (`free_rental_boards`) учитывается **отдельно** (Q1–Q4).
- **Политика отмен:** отмена клиентом доступна, если до старта осталось **более 10 минут** → место и прокат возвращаются; при **≤ 10 минут** отмена **заблокирована**.
- **Оплата офлайн** (наличные / перевод на карту) — приложение лишь фиксирует запись и показывает цену.
- **Период списка слотов по умолчанию — 7 дней (R-027).** По умолчанию клиенту показываются
  слоты на **ближайшие 7 дней** от текущего момента — это совпадает с горизонтом планирования
  расписания («на неделю вперёд»). Более длинный период доступен через **фильтр дат**. При
  отсутствии расписания на ближайшие дни показывается **empty state** («Пока нет доступных
  занятий»).
- **Отмена занятия мастерской / по форс-мажору (R-008).** Если занятие отменяет мастерская
  (организатор или из-за поломки оборудования), бронь на стороне клиента **не удаляется**, а
  получает статус **«Отменено мастерской»** с указанием причины; клиенту приходит
  **push-уведомление**. **Повторная запись на отменённый слот запрещена.** Само действие
  отмены слота выполняется в существующей инфраструктуре (вне скоупа клиентского приложения).

**Технические / платформенные**

- **Mobile-first**: клиентское приложение рассчитано на телефон (использование в мастерской и дома).
- Клиентское приложение взаимодействует с **существующим бэкендом через API**; слоты, программы и
  мастера доступны только для чтения.
- **Граница серверной интеграции (R-004).** Детали серверной реализации — транзакционность,
  SLA, маппинг сущностей на внутренние модели бэкенда — **вне скоупа клиентского приложения**.
  Существующий бэкенд — **black-box источник истины**: гарантия «**0 двойных броней**»
  обеспечивается на его стороне (атомарная проверка свободных мест при бронировании). Со стороны
  клиента граница закрывается **контрактом API**: приложение полагается на ответы бэкенда и
  корректно обрабатывает отказ, если мест уже нет.
- **Данные существующей инфраструктуры (R-015).** Это **учебный/тестовый проект**, исторических
  данных и легаси-схемы нет. **Канонической схемой считается контракт API** (OpenAPI в
  `01-analysis/api/`): бэкенд по условию отдаёт все поля контракта, включая прокатный тариф
  на инструменты и материалы, программу занятия, адрес мастерской и свободный прокатный фонд
  инструментов. **Миграция/backfill и дозаполнение исторических слотов вне скоупа.**

**Проект**

- **Срок** — к началу осеннего сезона, ~2 месяца (жёсткая цель).
- **Бюджет** ограничен → начинаем с самого нужного, «навороты» позже.

**Границы скоупа**

Скоуп — **клиентское мобильное приложение и API для него**. Управление расписанием, роли
мастера и владельца, админка — вне скоупа клиентского приложения.

| В скоупе клиентского приложения | Вне скоупа (существующая инфраструктура / не делаем сейчас) |
| :-- | :-- |
| Регистрация/вход клиента по имени и телефону | Роли Мастера и Владельца/Администратора, админ-экран Марины |
| Просмотр и **фильтрация** списка слотов (дата, программа, мастер) | Создание и редактирование расписания |
| Самостоятельная запись на слот с выбором проката | Отметка явки/неявки (определяет бэкенд) |
| Список бронирований: предстоящие и **история** | Онлайн-оплата |
| Отмена записи (**> 10 мин** до старта) | Автоматическая отмена по форс-мажору (действие в инфраструктуре) |
| **Оценка мастера** после занятия (1–5 звёзд) | Публичный показ рейтинга другим клиентам |
| **Статус / уровень лояльности** постоянного клиента | |
| Цена и фиксация записи (оплата офлайн) | |
| Push при отмене мастерской (Must); напоминание **~за 2 ч** (Should) | |

## Глоссарий

| Термин | Определение |
| :-- | :-- |
| **Мастер-класс / слот** | Запланированное групповое занятие: дата/время старта, программа, мастер, всего и свободно мест. |
| **Программа** | Вид занятия: новичковая лепка (потолок 6) или работа на гончарном круге (до 10 кругов). |
| **Группа** | Состав клиентов, записанных на один слот. |
| **Гончарный круг** | Рабочее место с кругом; всего 10 в мастерской. |
| **Прокат** | Выдача инструментов, фартука (и возможно материалов) мастерской клиенту без своего комплекта. |
| **Своё** | Клиент приходит со своими инструментами и фартуком; **не** расходует прокатный комплект. |
| **Отмена клиентом** | Доступна, если до старта **> 10 минут** — место и прокат возвращаются. При **≤ 10 минут** — заблокирована. |
| **OTP / SMS-код** | Одноразовый код подтверждения при входе по телефону (Q19). |
| **Deep link** | Схема `glina://bookings/{bookingId}` — переход в детали брони из push (MVP). |
| **Запись (бронь)** | Резерв места на конкретный слот: прокат да/нет, статус, оценка мастера (после занятия). |
| **Отмена** | Снятие записи клиентом (>10 мин до старта) или мастерской (форс-мажор → «Отменено мастерской» + push). |
| **Отменено мастерской** | Отмена занятия организатором или из-за поломки оборудования (выполняется в существующей инфраструктуре). На стороне клиента бронь сохраняется со статусом «Отменено мастерской» + причина; отправляется push-уведомление; повторная запись на отменённый слот недоступна. |
| **Администратор / владелец** | Марина — управляет расписанием и мастерами через существующую инфраструктуру (вне скоупа клиентского приложения). |
| **MVP** | Минимально жизнеспособный продукт: набор функций к старту осеннего сезона. |
