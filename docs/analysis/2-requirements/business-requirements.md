> Этап 2. Зачем бизнесу продукт: цели, метрики успеха, ограничения верхнего уровня.

> **Скоуп — клиентское мобильное приложение и API для него.** Бизнес-цели, обслуживаемые
> существующей инфраструктурой (расписание, работа мастеров, админка владельца), в этот
> список не входят.

> **Источник.** [`domain-description.md`](../1-elicitation/domain-description.md).

> **Роли в текущей поставке.** В текущую поставку входит только роль **Клиент**
> (самостоятельная запись на мастер-классы с мобильного приложения). Роли Мастера и
> Владельца/Администратора обеспечиваются существующей инфраструктурой.

## Ограничения верхнего уровня

- **Срок и бюджет.** Жёсткая цель — начало осеннего сезона (~2 месяца); бюджет ограничен → MVP-подход.
- **Оплата офлайн на старте.** Онлайн-оплата вне границ скоупа.
- **Mobile-first.** Клиентское приложение — с телефона; данные получает из существующего бэкенда через API.
- **Горизонт расписания.** По умолчанию 7 дней; расширенный период — через фильтр дат.
- **Бэкенд.** Black-box; гарантия «0 двойных броней» — на стороне бэкенда.
- **Вне скоупа.** Управление расписанием, роли мастера и владеля, онлайн-оплата, публичный рейтинг мастеров для других клиентов.
- **Порог отмены.** **10 минут** до `slot.start_at` — граница доступности отмены клиентом.
