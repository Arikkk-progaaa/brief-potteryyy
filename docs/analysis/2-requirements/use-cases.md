# Use cases / сценарии

> Этап 2. Сценарии использования. Обязательно прорабатываем **альтернативные потоки**.
>
> **Источник.** [`domain-description.md`](../1-elicitation/domain-description.md).
>
> **Скоуп: клиентское приложение и API для него.** Слоты, программы и мастера поступают
> из существующего бэкенда через API (read-only).

## UC-01: Просмотр расписания

- **Актёры:** Клиент
- **Предусловия:** Клиент открыл приложение, есть соединение с интернетом
- **Триггер:** Клиент ищет подходящий мастер-класс
- **Приоритет:** Must
- **Связанные FR:** FR-2–FR-5

### Основной поток
1. Клиент открывает главный экран приложения.
2. Система отображает список доступных слотов на ближайшие 7 дней (дефолтный период).
3. Каждый слот содержит: дату и время начала, программу, имя мастера, количество свободных мест, длительность.
4. Клиент может применить фильтры: по дате (календарь), по программе, по мастеру.
5. Система обновляет список слотов согласно фильтрам.
6. Клиент нажимает на интересующий слот — открывается детальная карточка с полной информацией (программа, мастер, свободные места, адрес, описание, опция проката инструментов).

### Альтернативные потоки
- **A1. Пустое расписание.** На ближайшие 7 дней нет ни одного слота → empty state: «Пока нет доступных занятий».
- **A2. Расширенный период.** Клиент задаёт более длинный диапазон через фильтр дат.

### Исключения / ошибки
- **E1. Нет слотов под условия фильтра.** Empty state с подсказкой изменить/сбросить фильтры.
- **E2. Нет интернета при запуске.** Заглушка «Нет соединения»; при наличии кэша — показ с пометкой «Данные могут быть устаревшими» (EC-01).

### Постусловия
- Клиент получил информацию о доступных слотах; данные расписания не изменяются.

## UC-02: Бронирование слота

- **Актёры:** Клиент
- **Предусловия:** Клиент авторизован, выбран слот со свободными местами
- **Триггер:** Клиент нажимает «Записаться» на детальной карточке слота
- **Приоритет:** Must
- **Связанные FR:** FR-6, FR-8–FR-11, FR-18

### Основной поток
1. Клиент нажимает кнопку «Записаться» на детальной карточке слота.
2. Система отображает форму бронирования:
   - Переключатель «Нужен прокат инструментов и фартука» (да/нет).
   - Стоимость (если прокат платный — итоговая сумма с учётом тарифа).
   - Кнопка «Подтвердить запись».
3. Клиент выбирает опцию проката и нажимает «Подтвердить запись».
4. Система отправляет запрос на бэкенд.
5. Бэкенд подтверждает бронирование (места есть).
6. Система отображает экран успешного бронирования с деталями: дата, время, программа, мастер, адрес, опция проката.
7. Система добавляет бронь в раздел «Мои записи».

### Альтернативные потоки
- **A1. Клиент не авторизован.** Перенаправление на экран входа/регистрации; после успешной авторизации — возврат на форму бронирования с сохранёнными данными (EC-16).

### Исключения / ошибки
- **E1. Нет свободных мест.** Бэкенд возвращает ошибку → «К сожалению, на это занятие больше нет мест. Выберите другой слот» (EC-05).
- **E2. Ошибка сети/сервера.** «Не удалось выполнить запись. Проверьте соединение и попробуйте снова»; данные формы не сбрасываются (EC-02, EC-06).
- **E3. Двойное нажатие.** Кнопка блокируется после первого нажатия до ответа бэкенда (EC-07).
- **E4. Слот уже начался или 0 мест.** Кнопка «Записаться» неактивна (EC-08, EC-09).
- **E5. Прокат недоступен.** Переключатель неактивен или предупреждение; бронирование возможно со своими инструментами (EC-21, EC-22).

### Постусловия
- Бронь создана (основной поток) или клиент уведомлён об ошибке.

## UC-03: Отмена бронирования клиентом

- **Актёры:** Клиент
- **Предусловия:** У клиента есть активная (предстоящая) бронь
- **Триггер:** Клиент решает отменить участие
- **Приоритет:** Must
- **Связанные FR:** FR-12–FR-14

### Основной поток
1. Клиент открывает раздел «Мои записи».
2. Система отображает список предстоящих броней.
3. Клиент выбирает бронь и нажимает «Отменить запись».
4. Система проверяет, что до начала занятия осталось **более 10 минут**.
5. Система показывает подтверждение: «Вы уверены, что хотите отменить запись?»
6. Клиент подтверждает отмену.
7. Система отправляет запрос на бэкенд.
8. Бэкенд подтверждает отмену.
9. Система отображает сообщение: «Запись отменена. Место освобождено».
10. Бронь перемещается в историю со статусом «Отменено клиентом».

### Альтернативные потоки
- **A1. Поздняя отмена (≤ 10 минут до начала).** Система отображает: «Отмена невозможна — до начала занятия осталось менее 10 минут. Если вы не можете прийти, свяжитесь с мастерской напрямую». Кнопка отмены заблокирована (EC-10).

### Исключения / ошибки
- **E1. Ошибка сети/сервера.** Сообщение об ошибке с предложением повторить.
- **E2. Бронь уже отменена или занятие прошло.** Кнопка отмены скрыта/неактивна (EC-11, EC-12).
- **E3. Токен авторизации истёк.** Экран входа; после повторной авторизации — возврат на предыдущий экран (EC-15).

### Постусловия
- Бронь отменена (основной поток) или клиент уведомлён о невозможности отмены.

## UC-04: Оценка мастера после занятия

- **Актёры:** Клиент
- **Предусловия:** Занятие завершилось (прошлое), клиент ещё не оценил мастера
- **Триггер:** Клиент хочет оценить мастера
- **Приоритет:** Must
- **Связанные FR:** FR-21

### Основной поток
1. Клиент открывает раздел «Мои записи» → вкладка «История».
2. Система отображает список прошедших занятий.
3. Напротив занятия без оценки отображается активная кнопка/звёзды «Оценить мастера».
4. Клиент нажимает на оценку (от 1 до 5 звёзд).
5. Система отправляет оценку на бэкенд.
6. Бэкенд подтверждает сохранение.
7. Система отображает «Спасибо за вашу оценку!» и блокирует возможность повторной оценки для этого занятия.

### Исключения / ошибки
- **E1. Ошибка отправки.** «Не удалось отправить оценку. Попробуйте позже»; кнопка остаётся активной (EC-03, EC-20).
- **E2. Занятие ещё не завершилось.** Кнопка оценки неактивна (EC-18).
- **E3. Оценка уже поставлена.** Повторная оценка заблокирована: «Вы уже оценили это занятие» (EC-19).

### Постусловия
- Оценка сохранена (основной поток) или клиент может повторить попытку.

## UC-05: Получение уведомления об отмене занятия мастерской (форс-мажор)

- **Актёры:** Система (push-уведомление) → Клиент
- **Предусловия:** У клиента есть активная бронь на слот, который мастерская отменила
- **Триггер:** Бэкенд фиксирует отмену слота мастерской (вне скоупа клиентского приложения)
- **Приоритет:** Must
- **Связанные FR:** FR-16, FR-17, FR-19

### Основной поток
1. Бэкенд фиксирует отмену слота мастерской.
2. Бэкенд отправляет push-уведомление клиенту.
3. Клиент получает уведомление: «Занятие [программа] [дата/время] отменено мастерской. Причина: [текст причины]».
4. Клиент открывает приложение по тапу на уведомление.
5. Система открывает детали брони.
6. Бронь отображается со статусом «Отменено мастерской» и указанием причины.
7. Кнопка «Записаться» на этот слот неактивна / скрыта.

### Альтернативные потоки
- **A1. Клиент не в сети.** При следующем входе бронь отображается со статусом «Отменено мастерской»; push доставляется при появлении соединения (EC-14).

### Постусловия
- Клиент уведомлён; бронь помечена как отменённая мастерской.

## UC-06: Получение напоминания о предстоящем занятии

- **Актёры:** Система (push-уведомление) → Клиент
- **Предусловия:** У клиента есть активная бронь на ближайшее время
- **Триггер:** До начала занятия остаётся заданный интервал (например, 2 часа)
- **Приоритет:** Should
- **Связанные FR:** FR-20

### Основной поток
1. Бэкенд (или клиентский планировщик) определяет, что до начала занятия осталось, например, 2 часа.
2. Система отправляет push-уведомление: «Напоминание: сегодня в [время] у вас занятие [программа] с мастером [имя]. Адрес: [адрес]».
3. Клиент тапает на уведомление — открывается деталь брони.

### Постусловия
- Клиент получил напоминание.
