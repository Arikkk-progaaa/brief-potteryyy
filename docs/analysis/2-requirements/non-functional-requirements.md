# Нефункциональные требования

> Этап 2. Как система должна работать: производительность, надёжность, безопасность,
> доступность, удобство. Частая ошибка ИИ — путать ФТ и НФТ.

> **Источник.** [`domain-description.md`](../1-elicitation/domain-description.md).

> **Скоуп — клиентское мобильное приложение и API для него.** НФТ относятся к клиентскому
> приложению; функции мастера/владельца/админки обеспечиваются существующей инфраструктурой.

| ID | Категория | Требование | Приоритет | Источник |
| :-- | :-- | :-- | :-- | :-- |
| NFR-1 | Удобство использования | Интерфейс клиента — **mobile-first**, рассчитан на использование с телефона в мастерской и дома. Skeleton-загрузчики при медленном соединении (EC-04). | Высокий | EC-04 |
| NFR-2 | Архитектура / Интеграция | Клиентское приложение взаимодействует с **существующим бэкендом через API**; слоты, программы и мастера доступны клиенту только для чтения. | Must | Описание домена → «Границы скоупа» |
| NFR-3 | Надёжность / Целостность данных | Гарантия **«0 двойных броней»** обеспечивается на стороне бэкенда; клиент корректно обрабатывает отказ при отсутствии мест. | Must | EC-05, EC-07 |
| NFR-4 | Надёжность / Целостность данных | Состояние записей согласовано: отмена клиентом (>10 мин) освобождает место; отмена мастерской не удаляет бронь, а меняет статус. | Высокий | FR-14, FR-16 |
| NFR-5 | Безопасность / Разграничение доступа | Клиент видит и управляет только **своими** бронированиями; при истечении токена — повторный вход с возвратом на экран (EC-15). | Высокий | EC-15, EC-16 |
| NFR-6 | Ограничения проекта | Запуск к **началу осеннего сезона** (~2 месяца) — жёсткая цель. | Must | Описание домена → «Проект» |
| NFR-7 | Ограничения проекта | **Ограниченный бюджет** → MVP-подход; онлайн-оплата откладывается. | Must | Описание домена → «Проект» |
| NFR-8 | Данные / Контракт | Учебный/тестовый проект без легаси-данных; **канонической схемой** считается контракт API. | Средний | Описание домена → R-015 |
| NFR-9 | Уведомления | Push-уведомление при отмене мастерской должно доставляться клиенту (EC-13, EC-14). | Must | UC-05 |
| NFR-10 | Уведомления | Напоминание о записи — push **за ~2 часа** до старта (Should). | Средний | UC-06 |
| NFR-11 | Сеть | При потере соединения во время операций (бронь, отмена, оценка) — понятные сообщения об ошибке без потери введённых данных где применимо (EC-02, EC-03). | Высокий | Edge cases |
