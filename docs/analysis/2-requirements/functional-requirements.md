# Функциональные требования

> Этап 2. Что система должна делать (функции, поведение).
> Не путать с нефункциональными ([non-functional-requirements.md](non-functional-requirements.md)).

Приоритеты по MoSCoW: **Must** — обязательно к запуску; **Should** — желательно при наличии времени.

> **Источник.** [`domain-description.md`](../1-elicitation/domain-description.md).

> **Скоуп — клиентское мобильное приложение и API для него.** Слоты, программы и мастера
> поступают из существующего бэкенда через API (read-only). Управление расписанием и ролями
> мастера/владельца — вне скоупа.

## Авторизация (клиент)

| ID | Требование | Приоритет | Источник |
| :-- | :-- | :-- | :-- |
| FR-1 | Система должна позволять клиенту регистрацию и вход по **имени и номеру телефона** с подтверждением **одноразовым кодом по SMS (OTP)** без пароля. | Must | Описание домена → «Границы скоупа» |

## Просмотр и фильтрация слотов (клиент)

| ID | Требование | Приоритет | Источник |
| :-- | :-- | :-- | :-- |
| FR-2 | Система должна показывать клиенту список предстоящих слотов **на ближайшие 7 дней** по умолчанию; более длинный период — через фильтр дат. Состав элемента: дата/время старта, программа, мастер, свободные места, длительность. | Must | Описание домена → R-027 |
| FR-3 | При отсутствии слотов в выбранном периоде система должна показывать empty state («Пока нет доступных занятий»). | Must | Описание домена → R-027 |
| FR-4 | Система должна позволять фильтровать список слотов по: **дате/периоду**, **программе**, **мастеру**. | Must | UC-01 |
| FR-5 | Система должна показывать карточку слота с параметрами занятия (программа, мастер, свободные места, длительность, адрес, описание, опция проката); программа и мастер — read-only из API. | Must | UC-01 |

## Запись на мастер-класс (клиент)

| ID | Требование | Приоритет | Источник |
| :-- | :-- | :-- | :-- |
| FR-6 | Система должна позволять клиенту записаться на выбранный слот (одно место на бронь). | Must | UC-02 |
| FR-8 | Система должна позволять при записи указать переключатель **«Нужен прокат инструментов и фартука»** (да/нет). | Must | UC-02 |
| FR-9 | Система должна ограничивать запись при отсутствии свободных мест; кнопка «Записаться» неактивна для слотов с 0 мест или уже начавшихся. | Must | EC-08, EC-09 |
| FR-10 | Система должна учитывать прокатный фонд инструментов/фартука: при выборе проката расходуется комплект; при «своё» — нет. | Must | Описание домена → «Глоссарий» |
| FR-11 | Система должна запрещать запись при отсутствии мест или прокатного инвентаря и корректно обрабатывать отказ API. | Must | EC-05, EC-21, EC-22 |

## Мои бронирования и отмены (клиент)

| ID | Требование | Приоритет | Источник |
| :-- | :-- | :-- | :-- |
| FR-12 | Система должна показывать клиенту список **своих** бронирований: предстоящие и история прошлых занятий с параметрами слота, опцией проката и статусом. | Must | US-07, US-09 |
| FR-13 | Система должна позволять клиенту отменить свою запись **до старта** занятия, если до начала осталось **более 10 минут**. | Must | UC-03 |
| FR-14 | При отмене (**> 10 минут** до `slot.start_at`) система должна освобождать место и прокатный комплект; бронь получает статус «Отменено клиентом». При **≤ 10 минут** до старта отмена **заблокирована**. | Must | UC-03, EC-10 |
| FR-16 | При отмене занятия **мастерской** (форс-мажор) бронь **не удаляется**, а получает статус **«Отменено мастерской»** с указанием причины. | Must | UC-05 |
| FR-17 | Система должна **запрещать повторную запись** на слот, отменённый мастерской. | Must | UC-05 |

## Edge cases

### Сеть и связь

| # | Edge Case | Ожидаемое поведение |
|---|-----------|---------------------|
| EC-01 | **Нет интернета при запуске приложения** | Заглушка «Нет соединения с интернетом». При кэше расписания — показ с пометкой «Данные могут быть устаревшими». |
| EC-02 | **Потеря соединения во время бронирования** | Индикатор загрузки с таймаутом; при превышении — «Не удалось выполнить запись. Проверьте соединение и попробуйте снова». Данные формы не сбрасываются. |
| EC-03 | **Потеря соединения при отправке оценки** | Сообщение об ошибке; кнопка оценки остаётся активной. |
| EC-04 | **Медленное соединение** | Skeleton-загрузчики при загрузке расписания и списка броней. |

### Бронирование и места

| # | Edge Case | Ожидаемое поведение |
|---|-----------|---------------------|
| EC-05 | **Бэкенд вернул ошибку «Нет мест»** | «К сожалению, на это занятие больше нет мест. Выберите другой слот». Возврат к расписанию. |
| EC-06 | **Бэкенд вернул 500/503 при бронировании** | «Временная ошибка сервера. Попробуйте позже» с возможностью повтора. |
| EC-07 | **Двойное нажатие «Записаться»** | Кнопка disabled + спиннер до ответа бэкенда. |
| EC-08 | **Запись на уже начавшийся слот** | Кнопка «Записаться» неактивна. |
| EC-09 | **Запись на слот с 0 мест** | Кнопка неактивна, «Мест нет». |

### Отмена брони

| # | Edge Case | Ожидаемое поведение |
|---|-----------|---------------------|
| EC-10 | **Отмена ≤ 10 минут до начала** | Отмена заблокирована: «Отмена невозможна — до начала занятия осталось менее 10 минут». |
| EC-11 | **Отмена уже отменённой брони** | Кнопка отмены скрыта/неактивна для «Отменено клиентом» / «Отменено мастерской». |
| EC-12 | **Отмена на прошедшее занятие** | Кнопка скрыта; в истории — «Посещено» или «Не явился» (определяет бэкенд). |

### Форс-мажор (отмена мастерской)

| # | Edge Case | Ожидаемое поведение |
|---|-----------|---------------------|
| EC-13 | **Отмена мастерской — клиент в приложении** | Статус «Отменено мастерской» + причина; повторная запись запрещена; push отправлен. |
| EC-14 | **Отмена мастерской — клиент офлайн** | При входе — статус и причина; push доставляется при появлении соединения. |

### Авторизация и данные

| # | Edge Case | Ожидаемое поведение |
|---|-----------|---------------------|
| EC-15 | **Токен авторизации истёк** | Экран входа; после повторной авторизации — возврат на предыдущий экран. |
| EC-16 | **Неавторизованный клиент пытается записаться** | Перенаправление на вход/регистрацию; после успеха — форма бронирования с сохранёнными данными. |
| EC-17 | **Расписание на 7 дней пустое** | Empty state: «Пока нет доступных занятий. Попробуйте изменить даты или загляните позже». |

### Оценка мастера

| # | Edge Case | Ожидаемое поведение |
|---|-----------|---------------------|
| EC-18 | **Оценка до окончания занятия** | Кнопка оценки неактивна для будущих и текущих занятий. |
| EC-19 | **Повторная оценка** | Заблокировано: «Вы уже оценили это занятие». |
| EC-20 | **Ошибка сохранения оценки** | Оценка не сохраняется; сообщение об ошибке; кнопка активна для повтора. |

### Прокат инструментов

| # | Edge Case | Ожидаемое поведение |
|---|-----------|---------------------|
| EC-21 | **Прокатный фонд исчерпан** | Переключатель неактивен или «Прокат временно недоступен»; запись со своими инструментами возможна. |
| EC-22 | **Ошибка «нет свободного инвентаря» при прокате** | «Прокат на эту дату недоступен. Запишитесь со своими инструментами или выберите другой слот». |

---

> **Вне скоупа.** Управление расписанием, роли мастера и владельца, онлайн-оплата,
> отметка явки/неявки, автоматическая отмена слота мастером — существующая инфраструктура
> или развитие; в клиентском приложении не реализуются.
