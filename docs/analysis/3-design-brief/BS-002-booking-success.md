## 1. Назначение и контекст

**Полноэкранное подтверждение успешной записи** — финальный шаг сценария «Записаться на
мастер-класс». Экран однозначно сообщает **«Вы записаны»**, показывает сводку брони,
напоминает об **офлайн-оплате** и предлагает следующий шаг.

BS-002 открывается **только после успешного оформления записи** на [SCR-004](SCR-004-booking.md).
Ошибки записи (нехватка мест, проката, гонка, сеть) обрабатываются **на SCR-004** и сюда
не доходят. Экран закрывает путь записи в рамках принципа «короткий путь к записи» (P2).

> **Тип — полноэкранный success**, не bottom sheet. Таб-бар скрыт до перехода «Готово» на SCR-002.

---

## 2. Пользователь и задача дизайна

- **Роль:** Клиент.
- **Цель пользователя:** получить однозначное подтверждение успеха, проверить ключевые
  параметры (когда, программа, сколько мест, своё/прокат, сумма) и понять способ оплаты (US-5, US-11).
- **Задача дизайна:** спокойное позитивное завершение без давления; мягко подвести к
  «Мои записи» или вернуть к каталогу занятий.

## 4. Контент на экране

| Элемент | Что показываем | Обязательность | Примечание |
| :-- | :-- | :-- | :-- |
| Иконка успеха | Форма + текст «готово» | Да | Не только цвет |
| Заголовок | **«Вы записаны»** | Да | Однозначное подтверждение |
| Дата и время | «Сб, 20 июл · 14:00» | Да | Когда занятие |
| Программа | Название (+ тип) | Да | На что записались |
| Мастер | «Мастер: …» | Да | — |
| Число мест | «Мест: N» | Да | Себя + гости |
| Своё / прокат | «N мест · из них M прокат» | Да | Лейблы «Своё» / «Прокат» |
| Итоговая цена | «Итого: … ₽» | Да | Read-only; не пересчитывается на экране |
| Напоминание об оплате | «Оплата на месте: наличные или перевод на карту.» | Да | foundations §6 |
| Блок push (опц.) | «Напомнить о занятии?» + «Включить» | Нет | Только **первая** успешная запись (FR-20) |
| CTA «Мои записи» | Primary | Да | → SCR-005 |
| CTA «Готово» | Secondary | Да | → SCR-002 |

**Чего на экране нет (явно):** грабер bottom sheet, автозакрытие по таймеру, «Поделиться»,
«Добавить в календарь», правило отмены (на SCR-006 / BS-003).

## 6. Компоненты и поведение

### 6.1 Блок успеха

- Иконка + заголовок «Вы записаны». Успех — формой/текстом, не только цветом
  ([foundations §3.2](00-foundations.md)).
- Тон спокойный, дружелюбный, без восклицательного давления ([foundations §6](00-foundations.md)).

### 6.2 Сводка брони

- Карточка «лейбл → значение»; один столбец (mobile-first).
- Итоговая цена — крупно; **read-only**, не пересчитывается на экране.
- При нулевом прокате строка про прокат может скрываться.

### 6.3 Кнопки

- **«Мои записи»** — основной CTA, во всю ширину → [SCR-005](SCR-005-my-bookings.md).
- **«Готово»** — вторичная → [SCR-002](SCR-002-slot-list.md).
- Тач-зоны ≥ 44–48 pt ([foundations §3.1](00-foundations.md)).

### 6.4 Запрос разрешения на push (FR-20 Should)

- **Только при первой успешной записи** клиента: после сводки — «Напомнить о занятии?» →
  по согласию **системный запрос push** ([foundations §8.1](00-foundations.md#81-уведомления-fr-19-fr-20)).
- **Единственное место** запроса в продукте; на [SCR-001](SCR-001-registration.md) не запрашивается.
- При последующих записях блок **не показывается**. Отказ не блокирует запись.

### 6.5 Поведение полноэкранного success

- Без автозакрытия по таймеру — клиент читает сводку и сам выбирает действие.
- Длинная сводка скроллится; кнопки остаются достижимы (фикс. внизу или в конце скролла).

## 9. Доступность и mobile-first

Ссылка на [foundations §3, §7](00-foundations.md). **Специфика этого экрана:**

- Крупные кнопки «Мои записи» и «Готово» — тач ≥ 44 pt.
- Иконка успеха + заголовок «Вы записаны» — смысл для screen reader текстом, не только иконкой.
- Ключевые числа (старт, места, цена) — контрастные.
- Layout устойчив к увеличению шрифта; при необходимости — скролл.
