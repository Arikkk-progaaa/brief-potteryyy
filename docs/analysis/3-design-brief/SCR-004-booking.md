## 1. Назначение и контекст

Экран собирает параметры брони (число мест, вариант «своё/прокат» на каждое место,
итоговую цену) и фиксирует запись. Это **самый сложный экран** клиентского сценария: на нём
пересекаются два независимых лимита (места в группе и прокатный фонд).

Экран — **последний шаг перед подтверждением** в потоке записи: список → карточка →
оформление → успех. Соблюдается правило **≤ 3 экранов до подтверждения** (P2): от
[SCR-002](SCR-002-slot-list.md) до [BS-002](BS-002-booking-success.md) клиент проходит
[SCR-003](SCR-003-slot-card.md) и этот экран.

> **Без хардкода.** Все числа (свободные места, прокатный фонд, цена, тариф проката,
> максимум мест на одну бронь) приходят из данных слота и **не зашиваются** в макет или тексты
> ([foundations P6](00-foundations.md#2-дизайн-принципы)). Правила двух лимитов и формула цены — **§6.5**.

## 2. Пользователь и задача дизайна

- **Роль:** Клиент.
- **Цель пользователя:** быстро записать себя и при необходимости гостей, выбрать «своё/прокат»
  на каждое место, увидеть итоговую цену и подтвердить бронь (US-5, US-6, US-7, US-8, US-11).
- **Задача дизайна:** собрать корректную бронь в рамках лимитов мест и прокатного фонда;
  показать итог read-only; защитить от двойного подтверждения; mobile-first — крупные элементы,
  минимум действий (P1, P2).

| Элемент | Что показываем | Обязательность | Примечание |
| :-- | :-- | :-- | :-- |
| Сводка слота | Дата/время, программа, мастер | Да | Read-only шапка |
| Доступность | «Свободно мест: N · прокат: M» | Да | Из данных слота |
| Счётчик «Число мест» | Степпер − / + , текущее значение | Да | Минимум 1; максимум — §6.5 |

| Строки «Своё / Прокат» | «Место 1 (вы)», «Место 2 (гость)»… + переключатель | Да | По одной на каждое место |

| Блок цены | Строки «Места», «Прокат» (если есть), **«Итого»** | Да | Read-only preview |
| Напоминание об оплате | «Оплата на месте: наличные или перевод на карту.» | Да | Под блоком цены |

**Чего на экране нет (явно):** поля имён гостей, онлайн-оплата, хардкод максимума мест (например «до 3»),
фиксированные лимиты программ.

> **Имена гостей не вводятся** — достаточно числа мест.

```
┌─────────────────────────────────────┐

├─────────────────────────────────────┤
│ Сб, 20 июл · 14:00                    │  сводка слота
│ Лепка для новичков · Анна            │  (read-only)
│ Свободно мест: N · прокат: M         │
│ ─────────────────────────────────── │
│ Число мест                           │  счётчик
│            [ − ]   2   [ + ]          │
│ Можно записать до K мест             │
│ ─────────────────────────────────── │
│ Своё / прокат для каждого места      │
│ Место 1 (вы)     [•Своё | Прокат]   │
│ Место 2 (гость)  [Своё |•Прокат]    │
│ Прокат выбрано: 1 из M               │
│ ─────────────────────────────────── │
│ Места: 2 500 ₽ × 2        5 000 ₽    │  блок цены
│ Прокат: 800 ₽ × 1           800 ₽    │  (если есть прокат)
│ Итого                     5 800 ₽    │  read-only, крупно
│ Оплата на месте: наличные/перевод    │
├─────────────────────────────────────┤
│ [        Записаться · 5 800 ₽      ] │  фикс. CTA
└─────────────────────────────────────┘
```

**Иерархия внимания:** сводка слота → счётчик мест → переключатели → итоговая цена → CTA.

## 6. Компоненты и поведение

### 6.1 Степпер «Число мест»

- Крупный степпер **«−» / «+»** (тач-зоны ≥ 44–48 pt, [foundations §3.1](00-foundations.md)).
- **Минимум — 1** (клиент всегда бронирует себя): на минимуме «−» disabled.
- **Максимум** — по правилу **§6.5** из данных. На максимуме «+» disabled; подпись
  поясняет лимит словами (число — из данных).
- При уменьшении числа мест лишние строки «своё/прокат» убираются с конца; при
  увеличении — добавляются со значением по умолчанию (§6.2).

### 6.2 Переключатели «Своё / Прокат» (на каждое место)

- Сегментный переключатель из двух опций; состояние дублируется не только цветом
  ([foundations §3.2](00-foundations.md)).
- **Значение по умолчанию для нового места:** «Прокат», если прокат ещё доступен;
  иначе «Своё».
- **Ограничение по прокату:** суммарно нельзя выбрать «Прокат» больше, чем доступно.
  При достижении границы опция «Прокат» на остальных местах **недоступна** с пояснением (§8).
- Подпись «Прокат выбрано: X из M» обновляется в реальном времени.

### 6.3 Блок итоговой цены (read-only)

- Строки **места** и, при наличии проката, **прокат**; **итог** крупно и контрастно.
- Пересчёт при изменении числа мест и переключателей. Итог дублируется на CTA.
- Под ценой — «Оплата на месте: наличные или перевод на карту.» ([foundations §6](00-foundations.md)).
- Финальная сумма на экране успеха — с сервера, без пересчёта клиентом.

### 6.4 Кнопка «Записаться» (нижний CTA)

- Во всю ширину, всегда видна ([foundations §4.1](00-foundations.md)).
- **Enabled**, когда бронь валидна по **§6.5**.
- **Disabled** при невалидной комбинации (поясняющая микрокопия, не «глухая» кнопка).
- **Submitting:** индикатор загрузки на CTA, форма блокируется, повторный тап невозможен.

### 6.5 Правила лимитов и цены (единый источник, FR-8–FR-11, FR-18)

**Лимиты (два независимых):**

| Лимит | Правило |
| :-- | :-- |
| Число мест | От 1 до меньшего из доступных ограничений (свободные места, потолок программы, максимум на бронь). **Фиксированный потолок в UI не зашивается.** |
| Прокат | Не больше доступного прокатного фонда. Считается из переключателей «Прокат». |

- **«Своё»** занимает место в группе, но **не** расходует прокатный фонд (FR-10).
- **«Прокат»** занимает место **и** расходует прокатный комплект (FR-10).
- При нулевом прокатном фонде запись возможна **только** со «Своим» на всех местах.

**Формула итоговой цены (preview в UI):**

```
Итого = цена_места × число_мест + тариф_проката × число_прокатных_мест
```

- Оба тарифа из данных слота. Эти правила — **единственное место** их описания на экране.

## 8. Тексты интерфейса (микрокопия)

Тон — спокойный, без вины ([foundations §6](00-foundations.md)). `[N]`, `[M]` — из актуальных данных.

| Контекст | Точный текст |
| :-- | :-- |
| Заголовок экрана | «Оформление записи» |
| Лейблы инвентаря | «Своё» / «Прокат» |
| E1 — нехватка мест | «Недостаточно мест. Свободно: [N]. Уменьшите число мест до [N].» |
| E2 — нехватка проката | «Недостаточно прокатных комплектов. Свободно: [M]. Выберите меньше проката или «Своё».» |
| E3 — места заняли параллельно | «Места уже заняты. Данные обновлены.» |
| E4 — сеть | «Не удалось загрузить. Проверьте соединение и попробуйте снова.» + «Повторить» |
| E4 — занятие отменено мастерской | «Занятие отменено мастерской» — запись невозможна |
| Офлайн-оплата | «Оплата на месте: наличные или перевод на карту.» |
| CTA | «Записаться» (+ итоговая сумма) |

## 9. Доступность и mobile-first

Ссылка на [foundations §3, §7](00-foundations.md). **Специфика этого экрана:**

- Крупный степпер и переключатели «Своё/Прокат» — тач-зоны ≥ 44–48 pt.
- Сценарий — **≤ 3 экранов** до BS-002 (P2).
- Итоговая цена — крупная, контрастная; продублирована на CTA.
- Состояния переключателей и disabled не только цветом.
- Screen reader: «Место 2, гость, прокат», «Увеличить число мест» и т. п.
- CTA не перекрывается клавиатурой; layout устойчив к увеличению шрифта.
