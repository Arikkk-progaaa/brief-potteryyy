## 1. Назначение и контекст

**SCR-002 «Список слотов» — главный экран приложения** и стартовая вкладка авторизованной
зоны. Показывает каталог предстоящих **групповых мастер-классов** и служит **единственной
точкой входа в сценарий записи**.

По умолчанию отображаются слоты на **ближайшие 7 дней**, отсортированные по ближайшей дате
старта и сгруппированные по дням. Более длинный период — через фильтр дат в [BS-001](BS-001-filters.md).
Заполненные слоты показываются с пометкой **«Мест нет»** (см. §6.5). Фильтр «только со
свободными местами» доступен в [BS-001](BS-001-filters.md), но **по умолчанию выключен**.

Отсюда клиент открывает карточку слота для записи или уточняет выдачу через фильтры.

---

## 2. Пользователь и задача дизайна

- **Роль:** Клиент.
- **Цель пользователя:** быстро просмотреть ближайшие занятия и выбрать подходящий
  мастер-класс (дата, программа, мастер, цена, наличие мест) — US-2, US-3.
- **Задача дизайна:** дать **быстрый выбор** — крупные карточки, контрастные ключевые числа,
  минимум мелкого текста (принципы P1, P2 в [foundations §2](00-foundations.md#2-дизайн-принципы)).

## 4. Контент на экране

### Уровень экрана

| Элемент | Что показываем | Обязательность | Примечание |
| :-- | :-- | :-- | :-- |
| Заголовок | «Занятия» | Да | — |
| Кнопка «Фильтры» | Иконка + текст «Фильтры» | Да | В хедере |
| Индикатор активных фильтров | Точка / бейдж | Нет | Когда применён ≥ 1 фильтр |
| Таб-бар | «Занятия» / «Мои записи» | Да | [foundations §4.2](00-foundations.md#42-таб-бар-авторизованная-зона) |

### Карточка слота (каждая строка списка)

| Элемент | Что показываем | Обязательность | Примечание |
| :-- | :-- | :-- | :-- |
| Дата и время | «Сб, 5 июля · 14:00» | Да | Крупно, контрастно |
| Программа | «Новичковая лепка», «Гончарный круг» и др. | Да | Название занятия |
| Мастер | «Мастер: Марина» | Да | — |
| Цена | «2500 ₽» | Да | Крупно, контрастно; из данных слота |
| Свободные места | «Свободно N из M» | Да | N и M из данных; ключевое число |
| Пометка «Мест нет» | Текст + иконка/форма | Нет | Только если мест нет; карточка некликабельна |

**Чего на экране нет (явно):** нижний CTA, экран профиля, хардкод лимитов программ (6 мест,
10 кругов) — только фактические числа из данных слота.

> Числовые лимиты **не зашиваются в тексты** — подставляются из данных («Свободно: N из M»).

## 5. Компоновка (wireframe)

Каркас — [foundations §4.1](00-foundations.md#41-базовый-каркас): хедер → скролл-список → таб-бар.
Нижнего CTA на этом экране нет.

```
┌─────────────────────────────────┐
│ Занятия          [⚙ Фильтры •]  │  ← хедер + индикатор фильтров
├─────────────────────────────────┤
│ ── Сб, 5 июля ──                 │  ← секция по дню (sticky при длинном списке)
│ ┌─────────────────────────────┐ │
│ │ [карточка слота — см. ниже] │ │
│ └─────────────────────────────┘ │
│ ┌─────────────────────────────┐ │
│ │ [карточка слота]            │ │
│ └─────────────────────────────┘ │
│              ⋮ (скролл)          │
├─────────────────────────────────┤
│ [ Занятия ]  [ Мои записи ]      │  ← таб-бар (foundations §4.2)
└─────────────────────────────────┘
```

**Карточка слота (вся область — одна тап-зона, если есть места):**

```
┌───────────────────────────────────────┐
│  Сб, 5 июля · 14:00                     │  ← дата/время (крупно)
│  Новичковая лепка                       │  ← программа
│  Мастер: Марина                         │  ← мастер
│  ─────────────────────────────────────  │
│  2500 ₽            Свободно 3 из 6       │  ← цена | места
└───────────────────────────────────────┘
```

**Иерархия внимания:** дата/время → программа → мастер → цена / места. Значения в wireframe условны.

## 6. Компоненты и поведение

### 6.1 Карточка слота

- **Вся карточка — единая тап-зона** при наличии свободных мест; переход в [SCR-003](SCR-003-slot-card.md).
- Видимая обратная связь на тап; доступное имя для screen reader.
- Карточки с явными отступами ([foundations §3.4](00-foundations.md#34-плотность)).

### 6.2 Кнопка «Фильтры»

- В хедере; открывает [BS-001](BS-001-filters.md).
- **Индикатор активных фильтров** — точка/бейдж при ≥ 1 применённом фильтре; не только цветом.

### 6.3 Сортировка и группировка

- Слоты сортируются **по ближайшей дате/времени старта**.
- При длинном списке — **заголовки-секции по дням** (sticky «дата»).

### 6.4 Pull-to-refresh

- Жест «потянуть вниз» обновляет список и **актуальную доступность мест**.
- Во время обновления — индикатор; контент не очищается без необходимости.

### 6.5 Слот без свободных мест

- Слот **показывается** с пометкой **«Мест нет»** (текст + иконка/форма) и **некликабелен**.
- Скрыть заполненные слоты можно фильтром «только со свободными местами» в
  [BS-001](BS-001-filters.md) (по умолчанию **выключен**).

### 6.6 Подгрузка списка

- В MVP список предстоящих слотов **грузится целиком** (горизонт по умолчанию — 7 дней;
  расширение — фильтром дат). Специальная пагинация не нужна.

## 9. Доступность и mobile-first

Ссылка на [foundations §3, §7](00-foundations.md). **Специфика этого экрана:**

- **Крупные карточки**, единая тап-зона — удобно одной рукой.
- **Ключевые числа контрастны и крупны**: места, цена, время старта.
- Пометка «Мест нет» и индикатор фильтров — не только цветом.
- Поддержка системного увеличения шрифта без поломки карточки.
- Карточки и «Фильтры» имеют доступные имена для screen reader.
