## 1. Назначение и контекст

Экран — **единственная точка входа** в приложение «Глина» для неавторизованного клиента.
Задача (job-to-be-done): **быстро попасть в приложение без пароля и лишних шагов**, чтобы
сразу перейти к списку мастер-классов и записаться на занятие.

Экран реализует принцип **минимального порога входа** ([foundations §2, P3](00-foundations.md#2-дизайн-принципы))
и обслуживает как **первичную регистрацию**, так и **повторный вход** по номеру телефона.
Отдельного выбора «регистрация / вход» нет — один экран, одна форма.

**Поток на экране (согласовано с API, Q19):**
1. Клиент вводит **имя** и **телефон** → «Продолжить» → SMS-код (`requestAuthCode`).
2. Клиент вводит **код из SMS (OTP)** — segmented-поле, длина из API → `verifyAuthCode`.
3. При успехе — переход в авторизованную зону на [SCR-002 «Список слотов»](SCR-002-slot-list.md).
   Для **нового** номера имя из шага 1 сохраняется через API профиля; для **известного** — шаг 3 пропускается.

Контекст использования — дома и в мастерской, mobile-first
([foundations §1](00-foundations.md#1-продукт-и-аудитория)).

## 2. Пользователь и задача дизайна

- **Роль:** Клиент.
- **Цель пользователя:** «Хочу быстро записаться на мастер-класс — войти без пароля и не
  заполнять длинную анкету» (US-1).
- **Задача дизайна:** провести вход за **минимум действий** (имя + телефон → «Продолжить»)
  и сразу отдать пользователя к каталогу занятий.

## 5. Компоновка (wireframe)

Каркас — [foundations §4.1](00-foundations.md#41-базовый-каркас). Таб-бар в НЗ **отсутствует**.
Нижний CTA фиксирован, **не перекрывается клавиатурой**.

```
┌───────────────────────────────┐
│  Глина                        │  ← хедер: название / приветствие
├───────────────────────────────┤
│                               │
│  Войдите, чтобы записаться     │  ← краткое пояснение
│  на мастер-класс               │
│                               │
│  Имя                          │
│  [____________________]       │  ← поле «Имя»
│                               │
│  Телефон                      │
│  [ +7 ___ ___-__-__ ]         │  ← поле «Телефон» (числовая клавиатура)
│                               │
│  Без пароля — вход по номеру   │  ← подсказка (caption)
│                               │
├───────────────────────────────┤
│  [      Продолжить      ]     │  ← фикс. нижний CTA
└───────────────────────────────┘
```

**Иерархия внимания:** пояснение → поля ввода → CTA «Продолжить»; подсказка «Без пароля» — вторичный текст.

## 6. Компоненты и поведение

### 6.1 Поле «Имя»

- Однострочное текстовое поле с лейблом; автофокус при первом открытии экрана (или после
  «Телефона» — на усмотрение реализации, главное — минимум переключений).
- Для **известного номера** имя может **предзаполняться** и остаётся редактируемым.

### 6.2 Поле «Телефон»

- Числовая / телефонная клавиатура, **маска** национального формата.
- По одному номеру выполняются и регистрация, и повторный вход.

### 6.3 Кнопка «Продолжить» (CTA)

- Во всю ширину; тач-зона — [foundations §3.1](00-foundations.md#31-тач-зоны).
- **Disabled:** пока имя или телефон не заполнены / не прошли формат-валидацию.
- **Loading:** после нажатия — индикатор, повторные тапы блокируются; по успеху → SCR-002.
- **Error:** сообщение под формой или общая заглушка с повтором (см. §8).

### 6.4 Повторный вход

- Клиент вводит ранее зарегистрированный телефон → сессия устанавливается без дополнительных
  экранов. Отдельного режима «вход» нет.

### 6.5 Запрос разрешения на push

- На этом экране **не запрашивается** — после первой успешной записи ([BS-002](BS-002-booking-success.md),
  [foundations §8.1](00-foundations.md#81-уведомления-fr-19-fr-20)).

## 7. Состояния экрана

Базовый паттерн — [foundations §5](00-foundations.md#5-сквозной-паттерн-состояний-экрана).

| Состояние | Что рисуем | Подсказка дизайнеру |
| :-- | :-- | :-- |
| **Content** | Поля «Имя», «Телефон», CTA «Продолжить», подсказка | Основной сценарий |
| **Валидация полей** | Подсветка невалидного поля + текст ошибки под полем | Не «ругается» во время набора |
| **Loading (отправка)** | CTA в состоянии Loading, поля заблокированы | Защита от двойной отправки |
| **Error (сеть / сервер)** | Сообщение + возможность повторить | Введённые данные **сохраняются** |
| **Error (бизнес)** | «Не удалось войти» — без технических деталей | Нейтральный тон |

## 8. Тексты интерфейса (микрокопия)

Тон — [foundations §6](00-foundations.md#6-tone-of-voice-и-микрокопия) (на «вы», без вины).

| Контекст | Точный текст |
| :-- | :-- |
| Пояснение | «Войдите, чтобы записаться на мастер-класс» |
| Лейбл «Имя» | «Имя» |
| Имя обязательно | «Укажите, как к вам обращаться» |
| Имя в недопустимом формате | «Проверьте имя — кажется, тут лишние символы» |
| Лейбл «Телефон» | «Телефон» |
| Телефон обязателен | «Введите номер телефона» |
| Телефон в неполном формате | «Похоже, номер введён не полностью» |
| Подсказка (без пароля) | «Без пароля — вход по номеру телефона» |
| CTA | «Продолжить» |
| Ошибка входа / сервера | «Не удалось войти. Попробуйте ещё раз» |
| Сетевая ошибка (общая) | «Не удалось загрузить. Проверьте соединение и попробуйте снова.» |

> **Пароля нет** — поле пароля отсутствует принципиально. **Минимум полей** — только имя и телефон.

## 9. Доступность и mobile-first

Ссылка на [foundations §3, §7](00-foundations.md). **Специфика этого экрана:**

- **Крупные поля и CTA** — тач-зоны ≥ 44–48 pt.
- **Телефонная клавиатура** для поля «Телефон».
- Лейблы и доступные имена у всех полей и кнопок (screen reader).
- Нижний CTA **не перекрывается** клавиатурой; форма скроллится при необходимости.
- Состояния disabled/ошибки дублируются не только цветом.
