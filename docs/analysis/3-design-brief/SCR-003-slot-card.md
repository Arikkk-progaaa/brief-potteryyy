## 1. Назначение и контекст

Экран показывает **все параметры одного мастер-класса (слота)**, чтобы клиент мог принять
решение о записи и перейти к оформлению. Это промежуточный шаг между списком занятий
([SCR-002](SCR-002-slot-list.md)) и оформлением записи ([SCR-004](SCR-004-booking.md)) —
вложенный экран без таб-бара (см. [foundations §4.2](00-foundations.md#42-таб-бар-авторизованная-зона)).

Контекст использования — дома и в мастерской, mobile-first
([foundations §1](00-foundations.md#1-продукт-и-аудитория)): клиенту нужно быстро считать
ключевые параметры (когда, где, программа, мастер, есть ли места и прокат, сколько стоит)
и решить, записываться ли.

> **Карта в MVP не используется.** Адрес мастерской — только текст
> ([foundations §4.5](00-foundations.md#45-адрес-мастерской)).

---

## 2. Пользователь и задача дизайна

- **Роль:** Клиент.
- **Цель пользователя:** убедиться, что занятие подходит (время, программа, мастер, цена,
  адрес мастерской), и понять, есть ли свободные места и прокатный фонд (US-4).
- **Задача дизайна:** дать полную и честную картину слота за один экран и предоставить
  один очевидный следующий шаг — «Записаться» — либо понятно объяснить, почему запись
  сейчас невозможна.

## 4. Контент на экране

| Элемент | Что показываем | Обязательность | Примечание |
| :-- | :-- | :-- | :-- |
| Дата и время старта | «Сб, 20 июля · 14:00» | Да | Крупно, контрастно |
| Название программы | «Лепка для новичков» и др. | Да | Блок «Программа» |
| Тип программы | Бейдж «Новичковая» / «На круге» | Да | Текст + форма, не только цвет |
| Длительность | «~2 ч 30 мин» | Нет | Только если значение есть в данных; иначе блок скрывается |
| Мастер | «Анна» | Да | Блок «Мастер»; без контактов |
| Свободные места | «Свободно 3 из 6» | Да | Ключевое число; влияет на CTA |
| Прокатный фонд | «Свободно 4 комплекта» | Да | Число из данных, без хардкода |
| Цена | «2 500 ₽ за место» | Да | Крупно, контрастно; за одно место |
| Адрес мастерской | «ул. Гончарная, 12» | Да | **Только текст**; без карты |
| Напоминание об оплате | «Оплата на месте: наличные или перевод на карту.» | Да | Под блоком цены |
| CTA «Записаться» | Primary, во всю ширину | Да | Фиксирован внизу |

**Чего на экране нет (явно):** интерактивная карта, «Открыть на карте», описание программы
(вне MVP), хардкод лимитов (6 мест, 12 комплектов) — только числа из данных слота.

> Потолок программы и прокатный фонд **не зашиваются в тексты** — отражаются через числа слота.

## 6. Компоненты и поведение

### 6.1 Кнопка «Назад»

- Возврат на [SCR-002](SCR-002-slot-list.md); тач-зона ≥ минимальной ([foundations §3.1](00-foundations.md#31-тач-зоны)).

### 6.2 Блоки параметров

- Карточки/строки «лейбл + значение». Ключевые числа (дата/время, цена, свободно мест) — крупные
  и контрастные ([foundations §3.2](00-foundations.md#32-контраст)).

### 6.3 Блок «Тип программы»

- Бейдж с текстовой подписью (не только цветом).

### 6.4 Блок «Длительность»

- Отображается при наличии значения в данных; при отсутствии — скрывается (без «прочерков»).

### 6.5 Блок «Прокатный фонд»

- Формулировка «Свободно N комплект(ов)»; число из данных.

### 6.6 Блок «Адрес»

- Только текст; без карты, без ссылок «Открыть на карте» ([foundations §4.5](00-foundations.md#45-адрес-мастерской)).

### 6.7 CTA «Записаться»

- **Enabled:** есть свободные места, занятие не отменено мастерской → переход на [SCR-004](SCR-004-booking.md).
- **Disabled:** нет мест — пометка **«Мест нет»**; занятие отменено — бейдж **«Занятие отменено мастерской»**.
  Тап не выполняет переход.

## 7. Состояния экрана

Базовый паттерн — [foundations §5](00-foundations.md#5-сквозной-паттерн-состояний-экрана).

| Состояние | Что рисуем | Подсказка дизайнеру |
| :-- | :-- | :-- |
| **Loading** | Скелетон блоков параметров + место под CTA | Форма будущего контента |
| **Content** | Все параметры слота + активный CTA «Записаться» | Основной сценарий |
| **Error** | Заглушка ошибки + «Обновить» | Тон из foundations §6 |
| **Нет свободных мест** | Все параметры; CTA **неактивна** с «Мест нет» | — |
| **Занятие отменено мастерской** | Параметры + бейдж «Занятие отменено мастерской» (+ причина, если есть); CTA **неактивна** | FR-17, UC-1 E4 |

## 8. Тексты интерфейса (микрокопия)

Тон — [foundations §6](00-foundations.md#6-tone-of-voice-и-общая-микрокопия).

| Контекст | Точный текст |
| :-- | :-- |
| Заголовок экрана | «Мастер-класс» (или название программы) |
| CTA неактивна (нет мест) | «Мест нет» |
| CTA неактивна (отмена мастерской) | «Занятие отменено мастерской» |

| Адрес | «Адрес: …» (текстовый блок) |
| Цена | «N ₽ за место» |
| Офлайн-оплата | «Оплата на месте: наличные или перевод на карту.» |
| CTA | «Записаться» |

> Числовые значения **не хардкодить** — подставляются из данных.

## 9. Доступность и mobile-first

Ссылка на [foundations §3, §7](00-foundations.md). **Специфика этого экрана:**

- Ключевые числа (дата/время, цена, свободно мест) — крупные и контрастные.
- CTA «Записаться» — во всю ширину, тач-зона ≥ минимальной.
- Состояния (тип программы, «Мест нет», «отменено мастерской») не передаются только цветом.
- Адрес мастерской доступен как текст (screen reader); карта не требуется.
- Layout не ломается при системном увеличении шрифта.

## 11. Зафиксированные UX-решения

| # | Решение | Почему |
| :-- | :-- | :-- |
| 1 | **Адрес — только текст**; карта и геолокация в MVP не используются | foundations §4.5 |
| 2 | **Длительность** — показывается только при наличии в данных; иначе блок скрывается | P6 |
| 3 | **Прокатный фонд** — число свободных комплектов из данных, без хардкода | FR-5 |
| 4 | **Описание программы** — вне MVP; на карточке название, тип и параметры слота | MVP scope |
| 5 | **Лимиты мест** — числа из данных; потолок программы не дублируется отдельной константой | P6 |
