## 1. Продукт и аудитория

**«Глина»** — клиентское мобильное приложение гончарной мастерской для самостоятельной записи
на групповые мастер-классы (~2–2,5 ч). Заменяет запись через Instagram Direct и ежедневник.

**Единственная роль — «Клиент».** Мастер и владелица работают через существующую инфраструктуру.
Слоты, программы и мастера — read-only из API. Оплата — **офлайн** (наличные / перевод).

**Контекст использования:** клиент пользуется телефоном **дома и в мастерской** (mobile-first,
NFR-1). Интерфейс — крупные элементы, высокий контраст, минимум шагов.

## 2. Дизайн-принципы

| # | Принцип | Источник | Смысл для макета |
|---|---------|----------|------------------|
| P1 | **Mobile-first** | NFR-1 | Крупные тач-зоны, высокий контраст, читаемость. |
| P2 | **Короткий путь к записи** | BR-2 | От списка до подтверждения — **≤ 3 экранов**. |
| P3 | **Минимальный порог входа** | FR-1 | Имя + телефон + подтверждение **SMS-кодом (OTP)** без пароля. |
| P4 | **Честность и спокойствие** | UC-1/UC-2 | Ошибки и правила объясняются понятно; штрафов на старте нет. |
| P5 | **Только свои данные** | NFR-5 | Клиент видит лишь свои бронирования. |
| P6 | **Данные из API** | NFR-8 | Лимиты мест, проката, цены — из ответа бэкенда, не хардкод в UI. |

## 3. Структурные токены (без бренда)

### 3.1 Тач-зоны
- Минимум **≥ 44–48 pt** по меньшей стороне.
- Основной CTA — во всю ширину, не перекрывается клавиатурой.

### 3.2 Контраст
- Не ниже **WCAG AA** (текст ≥ 4.5:1, крупный ≥ 3:1).
- Состояния не только цветом — дублировать текстом/иконкой.

### 3.3 Иерархия типографики
Заголовок экрана → заголовок секции → основной текст → подпись (caption).

### 3.4 Плотность
Одна колонка, карточки с явными отступами.

## 4. Каркас экрана и навигация

### 4.1 Базовый каркас

```
┌─────────────────────────────┐
│ Хедер (заголовок / назад)    │
├─────────────────────────────┤
│ Скролл-контент               │
├─────────────────────────────┤
│ Фикс. нижний CTA (если есть) │
└─────────────────────────────┘
│ Таб-бар (корневые экраны АЗ) │
└─────────────────────────────┘
```

### 4.2 Таб-бар (авторизованная зона)

Два раздела на корневых экранах:
- **Занятия** ([SCR-002](SCR-002-slot-list.md)) — стартовая вкладка.
- **Мои записи** ([SCR-005](SCR-005-my-bookings.md)).

Таб-бар скрывается на вложенных экранах и bottom sheet. Иконки с подписями.

### 4.3 Bottom Sheet (BS-001, BS-003)

- Высота по контенту, до ~90% экрана; длинный контент скроллится.
- Бэкдроп + закрытие по тапу вне (кроме критичных подтверждений).
- Swipe-to-close + грабер сверху.
- Анимация снизу вверх.

### 4.4 Карта навигации

Полная карта — [feature-list §3](../5-mobile-app-spec/feature-list.md).

### 4.5 Адрес мастерской

На экранах с деталями занятия ([SCR-003](SCR-003-slot-card.md),
[SCR-006](SCR-006-booking-details.md)) показывается **текстовый адрес** мастерской из API
(R-015). Интерактивная карта в MVP **не требуется**.

## 5. Сквозной паттерн состояний экрана

| Состояние | Отображение |
|-----------|-------------|
| **Loading** | Скелетон/шиммер |
| **Content** | Данные |
| **Empty** | Заглушка + подсказка + действие |
| **Error** | Ошибка + кнопка **«Обновить»** |

Специфика (disabled CTA «Записаться», бейдж «Отменено мастерской») — в экранных документах.

## 6. Tone of voice и микрокопия

**Тон:** простой, дружелюбный, на «вы», без давления.

| Контекст | Текст |
|----------|-------|
| Оплата | «Оплата на месте: наличные или перевод на карту.» |
| Лейблы инвентаря | «Своё» / «Прокат» |
| Правило отмены (общее) | «Отменить запись можно, если до начала занятия осталось **более 10 минут**. Место освободится для других.» |
| Отмена заблокирована | «Отмена невозможна — до начала занятия осталось менее 10 минут. Если вы не можете прийти, свяжитесь с мастерской напрямую.» |
| Отменено мастерской | «Занятие отменено мастерской» + причина |
| Empty слотов | «Пока нет доступных занятий» |
| Кнопка повтора | «Обновить» |
| Сетевая ошибка | «Не удалось загрузить. Проверьте соединение и попробуйте снова.» |

> Числовые лимиты (потолок программы, прокатный фонд) **не зашиваются в тексты** — из данных слота.

### 6.1 Снеки успеха

| Действие | Текст | Примечание |
|----------|-------|------------|
| Отмена | «Запись отменена. Место освобождено» | SCR-006 после BS-003 |
| Создание брони | — | Обратная связь — BS-002 |
| Фильтры | — | Обновлённый список на SCR-002 |

WCAG 2.1 AA: контраст, тач ≥ 44 pt, Dynamic Type, screen reader labels, не только цвет.

## 8. Сквозные функции

### 8.1 Уведомления (FR-19, FR-20)
- Push при отмене мастерской (Must).
- Напоминание о записи (Should) — **~за 2 часа** до старта (UC-06); запрос разрешения — после первой успешной записи на BS-002.

### 8.2 Безопасность в UI (NFR-5)
Только данные текущего клиента.

### 8.3 Deep links (push → экран)

| Схема | Назначение |
|-------|------------|
| `glina://bookings/{bookingId}` | Push при отмене мастерской (FR-19) → [SCR-006](SCR-006-booking-details.md) |

## 9. Глоссарий

| Термин | Значение |
|--------|----------|
| **Слот** | Мастер-класс: дата/время, программа, мастер, места, цена. |
| **Программа** | Новичковая лепка (≤6) или работа на круге (≤10 кругов). |
| **Прокат / Своё** | Комплект мастерской или свои инструменты и фартук. |
| **Отмена клиентом** | Доступна, если до старта **> 10 минут**; иначе заблокирована (EC-10). |
| **Отменено мастерской** | Форс-мажор; бронь сохраняется со статусом и причиной. |
