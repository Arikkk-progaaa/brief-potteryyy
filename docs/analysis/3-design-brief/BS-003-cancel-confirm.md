## 1. Назначение и контекст

Шторка открывается, когда клиент инициирует **отмену предстоящей записи** на мастер-класс и
должен осознанно подтвердить действие.

Назначение:
- Объяснить **последствия** до выполнения: место и прокатный инвентарь освободятся.
- Получить **явное подтверждение** и защитить от случайного нажатия.

> Шторка открывается только при `can_cancel = true` (> **10 минут** до старта, UC-03, FR-14).
> При ≤ 10 минут CTA «Отменить» на SCR-006 disabled с текстом EC-10.

Шторка открывается поверх [SCR-006 · Детали брони](SCR-006-booking-details.md). Результат
отображается на **родительском экране** (обновлённый статус + снек), не внутри шторки.

> Шторка **не открывается** для записей «Отменено мастерской» — форс-мажор обрабатывается на SCR-006 (UC-4).

---

## 2. Пользователь и задача дизайна

- **Роль:** Клиент.
- **Цель пользователя:** отказаться от участия в занятии, понимая, что место освободится (US-08).
- **Задача дизайна:** простое подтверждение «Вы уверены?»; безопасное действие
  по умолчанию («Не отменять»); явное деструктивное подтверждение.

---

## 3. Навигация

**Вход:** кнопка **«Отменить»** на [SCR-006](SCR-006-booking-details.md) — только если запись
активна и занятие ещё не началось.

**Выход:**

| Действие | Результат |
| :-- | :-- |
| **«Подтвердить отмену»** | Закрытие + возврат на SCR-006 с обновлённым статусом и снек-итогом |
| **«Не отменять»** / закрытие (swipe / бэкдроп) | SCR-006 **без изменений** |

Для **критичного подтверждения:** закрытие по swipe / бэкдроп = **отказ**; отмена невозможна
без явного тапа «Подтвердить отмену».

## 6. Компоненты и поведение

### 6.1 Пояснение

«Место освободится и станет доступно другим.» — нейтральный тон.

### 6.2 Кнопки

- **«Подтвердить отмену»** — деструктивное, **менее акцентное**; смысл дублируется текстом.
- **«Не отменять»** — безопасное, **основное/акцентное**; не «ловушка» для случайного тапа.
- Тач-зоны ≥ [foundations §3.1](00-foundations.md#31-тач-зоны).

### 6.3 Выполнение (loading)

После «Подтвердить отмену» — блокировка повторных тапов до завершения; индикатор на кнопке.

### 6.4 Исходы (логика UI)

| Исход | UI |
| :-- | :-- |
| Успех | Закрыть шторку → SCR-006; снек «Запись отменена. Место освобождено» |
| Ошибка сети | Текст из foundations + повтор; шторка открыта |
| Занятие уже началось / ≤10 мин | SCR-006: CTA disabled (UC-03) |

| Состояние | Что рисуем |
| :-- | :-- |

| **Выполнение** | Loading на кнопке подтверждения |

## 8. Тексты интерфейса

| Контекст | Текст |
| :-- | :-- |
| **Общее правило** | «Отменить запись можно, если до начала занятия осталось **более 10 минут**. Место освободится для других.» |
| **Подтверждение** | «Вы уверены, что хотите отменить запись?» |
| **Успех** | «Запись отменена. Место освобождено» |
| **Заголовок** | «Отменить запись?» |
| **Подтверждение** | «Подтвердить отмену» |
| **Отказ** | «Не отменять» |
| **Ошибка сети** | «Не удалось загрузить. Проверьте соединение и попробуйте снова.» |

**Запрещено в копирайте:** угрозы штрафов, вина клиента.

---

## 9. Доступность

- Защита от случайного подтверждения: только явный тап «Подтвердить отмену».
- «Не отменять» — понятное доступное имя; визуально предпочтительнее деструктивной кнопки.
- Ранняя/поздняя — текст + форма/иконка, не только цвет.
- Длинный контент шторки скроллится ([foundations §4.3](00-foundations.md#43-bottom-sheet-bs-001-bs-003)).
- CTA достижимы без скролла или с фиксированной нижней зоной кнопок.
