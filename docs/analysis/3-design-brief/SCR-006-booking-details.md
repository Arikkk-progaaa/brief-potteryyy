## 1. Назначение и контекст

Экран показывает **полную информацию об одной записи (брони)** клиента и служит точкой
**запуска её отмены**. Вложенный экран: открывается из [SCR-005 «Мои записи»](SCR-005-my-bookings.md)
тапом по карточке **или по push-уведомлению** при отмене занятия мастерской (FR-19, UC-4).

Две задачи:
1. **Просмотр.** Все параметры брони и текущий статус: когда и на какую программу записан клиент,
   мастер, адрес мастерской, места, своё/прокат, итоговая цена (офлайн-оплата).
2. **Отмена.** Единственное действие над записью — отменить участие **до старта занятия**.
   Подтверждение отмены — в шторке
   [BS-003](BS-003-cancel-confirm.md).

Контекст — дом и мастерская ([foundations §1](00-foundations.md#1-продукт-и-аудитория)): статус
и ключевые числа читаемы с первого взгляда; деструктивное действие защищено подтверждением.

> **Карта маршрута в MVP нет** — только **текстовый адрес** мастерской ([foundations §4.5](00-foundations.md#45-адрес-мастерской)).

## 2. Пользователь и задача дизайна

- **Роль:** Клиент.
- **Цель пользователя:** убедиться в деталях записи; при необходимости отменить её до начала,
  понимая последствия; при форс-мажоре — увидеть статус «Отменено мастерской» и причину (US-10, US-12, US-13).
- **Задача дизайна:** read-only детали с читаемым статусом; одна деструктивная CTA «Отменить»
  с disabled-состояниями и пояснениями; без случайной отмены (подтверждение в BS-003).

## 4. Контент на экране

| Элемент | Что показываем | Обязательность | Примечание |
| :-- | :-- | :-- | :-- |
| Бейдж статуса | «Активна», «Отменена», «Поздняя отмена», **«Отменено мастерской»**, «Прошедшая» | Да | Текст + форма |
| Когда | Дата и время старта | Да | Блок «Когда» |
| Программа | Название + тип (новичковая / на круге) | Да | Блок «Программа» |
| Мастер | «Мастер: …» | Да | — |
| Адрес | Текст адреса мастерской | Да | **Только текст**; без карты |
| Места | «Мест: N» | Да | Себя + гости |
| Инвентарь | «N прокат · M своё» | Да | «Своё» / «Прокат» |
| Цена | Итоговая сумма | Да | Серверный итог; не пересчитывается |
| Оплата | «Оплата на месте: наличные или перевод на карту.» | Да | Под ценой |
| Записано | Дата и время оформления | Да | Служебная метка |
| Отменена | Дата отмены | Нет | Только если запись отменена |
| Причина | Текст причины | Нет | Только при «Отменено мастерской» |
| Подсказка правила отмены | Краткая формулировка | Нет | При активной CTA «Отменить» |
| CTA «Отменить» | Деструктивная, во всю ширину | Да | Фиксирован внизу |

**Чего на экране нет (явно):** интерактивная карта, CTA «Записаться снова» на отменённый слот,
частичная отмена мест, фиксированный числовой порог ранней/поздней отмены.

## 6. Компоненты и поведение

### 6.1 Блоки деталей

Read-only секции в одну колонку ([foundations §3.4](00-foundations.md#34-плотность)).
Ключевые числа (старт, цена, места) — крупные и контрастные.

### 6.2 Бейдж статуса

- **Активна** — запись активна, старт в будущем.
- **Прошедшая** — активна, старт уже наступил.
- **Отменена** — ранняя отмена клиентом.
- **Поздняя отмена** — поздняя отмена клиентом.
- **Отменено мастерской** — форс-мажор (FR-16); под бейджем — причина.

Смысл — иконка + текст, не только цвет.

### 6.3 Кнопка «Отменить» (нижний CTA)

- **Enabled** — запись активна **и** занятие ещё не началось (FR-13). Тап → [BS-003](BS-003-cancel-confirm.md).
- **Disabled (не скрывается)** — занятие уже началось; запись уже отменена, в т.ч. мастерской.
  Обязательное **короткое пояснение** рядом или под кнопкой.
- **Отмена только целиком** — частичная отмена мест в MVP не поддерживается.

### 6.4 Адрес мастерской

Текстовый блок из данных ([foundations §4.5](00-foundations.md#45-адрес-мастерской)).
Без интерактивной карты в MVP.

### 6.5 Push-deeplink (FR-19, UC-4)

При открытии по push после форс-мажора экран показывает бейдж «Отменено мастерской», причину
и disabled CTA «Отменить» с пояснением «Запись уже отменена» / «Занятие отменено мастерской».
CTA «Записаться снова» **не показываем** (FR-17).

### 6.6 Подсказка правила отмены

При активной CTA — **краткая** формулировка из [foundations §6](00-foundations.md#6-tone-of-voice-и-общая-микрокопия)
Полный текст — в [BS-003](BS-003-cancel-confirm.md). Порог **10 минут** (UC-03, FR-14).

## 7. Состояния экрана

Базовый паттерн — [foundations §5](00-foundations.md#5-сквозной-паттерн-состояний-экрана).

| Состояние | Что рисуем | Подсказка дизайнеру |
| :-- | :-- | :-- |
| **Loading** | Скелетон блоков деталей и бейджа | — |
| **Content (активная)** | Полные детали + enabled CTA «Отменить» + краткая подсказка | Основной сценарий |
| **Content (отменена клиентом)** | Бейдж «Отменена» / «Поздняя отмена», дата отмены; CTA disabled «Запись уже отменена» | — |
| **Content (отменено мастерской)** | Бейдж «Отменено мастерской», причина; CTA disabled | UC-4, US-13 |
| **Content (занятие началось)** | Бейдж «Прошедшая»; CTA disabled «Занятие уже началось — отменить нельзя» | UC-2 E1 |
| **Error** | Заглушка + «Обновить» | Тон из foundations §6 |

Empty state неприменим — экран всегда про существующую бронь.

## 9. Доступность и mobile-first

Ссылка на [foundations §3, §7](00-foundations.md). **Специфика этого экрана:**

- Бейдж статуса — текст + иконка; «Отменено мастерской» семантически отделён от клиентской отмены.
- CTA «Отменить» — во всю ширину, тач ≥ 44 pt; disabled-состояние доступно screen reader.
- Отмена не по одному тапу — только через [BS-003](BS-003-cancel-confirm.md).
- Адрес — текстовый fallback без карты (доступность).
