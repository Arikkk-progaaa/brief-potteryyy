## 1. Назначение и контекст

Экран даёт клиенту **контроль над своими мастер-классами** — отдельно **предстоящие** и
**прошедшие** записи. Это домашняя точка вкладки **«Мои записи»** таб-бара
(см. [foundations §4.2](00-foundations.md#42-таб-бар-авторизованная-зона)): клиент с одного
взгляда понимает, на что записан, видит статус каждой брони и переходит к деталям или отмене.

Контекст использования — дома и в мастерской, mobile-first
([foundations §1](00-foundations.md#1-продукт-и-аудитория)): крупные карточки, читаемые даты
и статусы, минимум шагов до действия.

---

## 2. Пользователь и задача дизайна

- **Роль:** Клиент.
- **Цель пользователя:** быстро увидеть предстоящие и прошедшие записи на мастер-классы,
  убедиться в деталях ближайшего занятия, при необходимости открыть детали или отменить;
  при отсутствии записей — понять, как записаться (US-9, US-13).
- **Задача дизайна:** разделить записи на «Предстоящие» и «Прошедшие»; показать статусы
  текстом и формой; дать быстрый переход к деталям одним тапом по карточке.

## 4. Контент на экране

### Уровень экрана

| Элемент | Что показываем | Обязательность | Примечание |
| :-- | :-- | :-- | :-- |
| Заголовок | «Мои записи» | Да | — |
| Переключатель секций | «Предстоящие» / «Прошедшие» | Да | По умолчанию — «Предстоящие» |
| Таб-бар | «Занятия» / «Мои записи» | Да | [foundations §4.2](00-foundations.md#42-таб-бар-авторизованная-зона) |

### Карточка записи

| Элемент | Что показываем | Обязательность | Примечание |
| :-- | :-- | :-- | :-- |
| Дата и время | «сб, 12 сен · 14:00» | Да | Крупно, основной ориентир |
| Программа | «Лепка для новичков» и др. | Да | — |
| Мастер | «Мастер: Анна» | Да | — |
| Места и инвентарь | «2 места · 1 прокат» / «1 место · своё» | Да | «Своё» / «Прокат» |
| Итоговая цена | «4500 ₽» | Да | Серверный итог; клиент не пересчитывает |
| Бейдж статуса | «активна», «отменена», «поздняя отмена», **«Отменено мастерской»** | Да | Текст + форма, не только цвет |
| Причина отмены | «Поломка печи» и др. | Нет | Только при «Отменено мастерской», если причина есть |

**Чего на экране нет (явно):** нижний CTA, действия отмены на карточке (отмена — на SCR-006),
записи других клиентов.

### Правило разделения списка

- **Предстоящие** — **только активные** записи с датой старта в будущем.
- **Прошедшие** — записи с датой старта в прошлом **и все отменённые** (в т.ч. с будущим стартом).
- Предстоящие — по возрастанию даты; прошедшие — по убыванию (свежие сверху).

> Карточка — **сводка**; полная детализация — на [SCR-006](SCR-006-booking-details.md).

## 5. Компоновка (wireframe)

Каркас — [foundations §4.1](00-foundations.md#41-базовый-каркас): хедер → скролл-список → таб-бар.

```
┌──────────────────────────────┐
│  Мои записи                  │  ← заголовок экрана
│  [ Предстоящие | Прошедшие ] │  ← сегмент-переключатель
├──────────────────────────────┤
│ ┌──────────────────────────┐ │
│ │ сб, 12 сен · 14:00       │ │  ← дата/время старта
│ │ Лепка для новичков       │ │  ← программа
│ │ Мастер: Анна             │ │
│ │ 2 места · 1 прокат       │ │
│ │ 4500 ₽          [активна]│ │  ← бейдж статуса
│ └──────────────────────────┘ │
│ ┌──────────────────────────┐ │
│ │ вс, 7 сен · 11:00        │ │
│ │ Работа на круге          │ │
│ │ Мастер: Игорь            │ │
│ │ 1 место · своё           │ │
│ │ [Отменено мастерской]    │ │
│ │ Поломка печи             │ │  ← причина (если есть)
│ └──────────────────────────┘ │
├──────────────────────────────┤
│  Занятия · Мои записи        │  ← таб-бар (2 вкладки)
└──────────────────────────────┘
```

**Иерархия внимания:** дата/время и цена — наиболее контрастные элементы карточки.

## 6. Компоненты и поведение

### 6.1 Переключатель «Предстоящие / Прошедшие»

- Сегмент-контрол вверху **или** sticky-секции (выбор — за дизайнером).
- По умолчанию открыты **Предстоящие**. Активная вкладка — текст + состояние, не только цвет.

### 6.2 Карточка записи

- Кликабельна целиком (тач-зона ≥ [foundations §3.1](00-foundations.md#31-тач-зоны)); тап → [SCR-006](SCR-006-booking-details.md).
- Одна колонка, явные отступы.

### 6.3 Бейдж статуса

- Текст + форма/иконка — [foundations §3.2](00-foundations.md#32-контраст). Тексты — §8.

### 6.4 Причина отмены мастерской

- Под бейджем «Отменено мастерской» — одна-две строки причины (если заполнено).

### 6.5 Pull-to-refresh

- Обновляет список (актуальные статусы после отмены или форс-мажора).

## 9. Доступность и mobile-first

Ссылка на [foundations §3, §7](00-foundations.md). **Специфика этого экрана:**

- **Крупные карточки** с большой тач-зоной — удобно в мастерской и дома.
- **Читаемые статусы:** бейдж дублирует смысл текстом и формой; «Отменено мастерской» отличимо
  от клиентской отмены.
- **Даты/время:** день недели + дата + время без двусмысленности.
- Переключатель секций и карточки имеют доступные имена для screen reader.

## 11. Зафиксированные UX-решения

| # | Решение | Почему |
| :-- | :-- | :-- |
| 1 | **Отменённые записи — всегда в «Прошедшие»** (в т.ч. с будущим стартом); «Предстоящие» — только активные с будущей датой | FR-12 |
| 2 | **Сортировка:** предстоящие — по возрастанию даты; прошедшие — по убыванию | P2 |
| 3 | **UI «Программа» / «Мастер»** — терминология домена «Глина» | Согласование с SCR-003 |
| 4 | **Итог цены** — серверный, клиент не пересчитывает | FR-18 |
| 5 | **Отмена только с экрана деталей** — на карточке списка действий отмены нет | UC-2 |
